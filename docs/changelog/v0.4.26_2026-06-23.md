# Changelog v0.4.26 - 2026-06-23

## Clannad47

### 修复 Windows 桌面端工作区创建转圈及代理请求阻塞

#### 问题描述

- 现象 1：在 Windows 桌面端创建工作区时，即使实际初始化已完成，界面仍持续显示转圈等待状态，用户体验异常。
- 现象 2：桌面端预览代理在 Windows 上转发 POST/PUT 请求时，后端报错或响应异常，导致文件同步和预览功能受阻。
- 原因 1：workspace 初始化状态没有在不需要 Python/Node 运行时的情况下立即标记完成，导致前端 spinner 持续等待。
- 原因 2：`local_preview_server.py` 直接将 `self.rfile` 传给 urllib，在 Windows 上无法正确读取请求体。

#### 改动内容

- 优化 workspace 创建流程：当不需要 Python/Node 运行时，立即标记初始化完成，避免不必要的 pending 状态和 spinner。
- 修复 Windows 代理请求体读取：显式按 `Content-Length` 读取请求体后再转发给 urllib，解决 Windows 上的 body blocking 问题。
- 增强 Windows 安装脚本：增加包装目录处理和跨设备安全复制逻辑，提升安装健壮性。
- 增加 `AIASYS_AGENT_MODE` 自动审批：桌面端 agent 自动化场景下可自动通过确认，减少人工干预。
- 侧边栏增加版本详情显示：支持查看后端与前端版本信息。

#### 文件变更

- `apps/backend/app/services/workspace_registry.py`: 优化 workspace 初始化状态机。
- `apps/web/scripts/committed/local_preview_server.py`: 修复 Windows POST/PUT body 转发。
- `apps/desktop/src/main.cjs`: workspace spinner 优化与 agent mode 自动审批。
- `apps/desktop/src/service-manager.cjs`: Windows 代理与运行时管理增强。
- `apps/desktop/scripts/install.cjs`: 增加安装目录处理与跨设备安全复制。
- `apps/desktop/build/installer.nsh`: NSIS 安装器包装目录处理。
- `apps/web/src/components/layout/WorkspaceSidebar/SidebarHeader.tsx`: 增加后端/前端版本详情入口。
