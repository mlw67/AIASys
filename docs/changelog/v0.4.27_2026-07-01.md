# Changelog v0.4.27 - 2026-07-01

## WWK-TJU

### 修复 Agent 执行请求中的 Thinking 配置不生效

#### 问题描述
- 现象：Workspace 页面可以切换 Thinking 开关和 low / medium / high 强度，但发起 Agent 对话后，请求级配置没有进入最终 LLM runtime request options。
- 原因：前端提交链路没有把 `thinkingEnabled` / `thinkingEffort` 继续传给 `runAgentStream`，后端 `execute_stream()` 也没有把 `thinking_enabled` / `thinking_effort` 传给 `_get_config()`。

#### 改动内容
- 在 Workspace 提交流程中继续透传 Thinking 开关和强度配置。
- 在 Agent 执行链路中将请求级 Thinking 配置传入 `_get_config()`，使动态模型配置覆盖逻辑生效。
- 增加后端回归测试，覆盖 `execute_stream()` 参数透传和最终 `LlmRequestOptions` 生效。

#### 文件变更
- `apps/web/src/pages/WorkspacePage/hooks/useCodeExecutor/useExecutionSubmit.ts`: 透传 Thinking 配置到 `runAgentStream`。
- `apps/backend/app/services/agent/mixins/execution.py`: 透传请求级 Thinking 配置到 `_get_config()`。
- `apps/backend/tests/test_agent_thinking_request_options.py`: 新增请求级 Thinking 配置回归测试。
