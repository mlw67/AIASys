### 修复同一工作区新建第二个会话时报 API Error: 400 的问题

#### 问题描述

- 现象：在当前工作区新建第二个会话后发送首条消息，后端返回 `API Error: 400`，旧会话可以正常对话。
- 原因：`GET /api/sessions/available-draft` 返回草稿会话时没有按工作区隔离。前端用当前工作区的 `workspace_id` 去调用 `POST /api/agent/execute/stream`，但拿到的草稿实际绑定在另一个工作区，触发 `session_id 与 workspace_id 不匹配` 的 400 校验。

#### 改动内容

- `GET /api/sessions/available-draft` 增加 `workspace_id` query 参数，只返回属于指定工作区的草稿。
- 前端 `requestAvailableDraftId` 调用时传入当前工作区 ID。
- 删除 `apps/backend/app/api/routes/sessions.py` 中未使用的 `_find_available_draft_for_user` 死代码，避免测试与实现脱节。

#### 文件变更

### 修复 prepare-runtime 误删运行时依赖导致打包产物运行失败的问题

#### 问题描述

- 现象：桌面端打包后的应用在代码执行、知识库等功能上可能报 `ModuleNotFoundError`。
- 原因：`pruneDevDependencies()` 把 `pip` 和 `wrapt` 列入开发依赖并删除。`pip` 被代码执行工具与运行时环境接口使用；`wrapt` 是 graph 可选依赖 `graspologic -> gensim -> smart-open -> wrapt` 的传递依赖。

#### 改动内容

- 从 `devPackages` 列表中移除 `pip` 和 `wrapt`，并补充注释说明保留原因。

#### 文件变更

- `apps/desktop/scripts/prepare-runtime.cjs`

### 修复异步路由中同步 I/O 阻塞事件循环的问题

#### 问题描述

- 现象：大文件上传、文件复制/移动、Node 运行时安装/切换等操作期间，Agent SSE 流、终端 WebSocket、其他 API 请求出现卡顿或超时。
- 原因：这些操作直接在 `async def` 路由中调用同步的 `shutil` 或 `subprocess.run`，长时间占用主事件循环。

#### 改动内容

- 文件上传的写盘操作改为 `asyncio.to_thread`。
- 文件移动/复制的 `shutil.move/copytree/copy2` 改为 `asyncio.to_thread`。
- Node 运行时安装/切换/默认设置/卸载/远程版本列表改为 `asyncio.to_thread`。

#### 文件变更

- `apps/backend/app/api/routes/workspaces_resources_files.py`
- `apps/backend/app/api/routes/files_core.py`
- `apps/backend/app/api/routes/runtime_envs.py`

### 修复静默吞异常导致状态误判的问题

#### 问题描述

- 现象 1：文件读取失败时 `_is_binary_file()` 返回 `False`，可能把二进制文件误判为文本，后续读取输出乱码或崩溃。
- 现象 2：`agent_config` 读取本地覆盖文件失败时返回 `True`，导致配置系统误判为存在本地覆盖。
- 现象 3：GraphRAG 路由的 LLM 未配置异常缺少 `from e`，异常链断裂。

#### 改动内容

- `_is_binary_file()` 读取失败时保守返回 `True`。
- `_has_mode_local_config()` 读取失败时保守返回 `False`。
- GraphRAG 路由的 HTTPException 补充 `from e`。

#### 文件变更

- `apps/backend/app/agents/tools/file_tools.py`
- `apps/backend/app/services/agent_config/service.py`
- `apps/backend/app/graphrag/api/routes.py`
